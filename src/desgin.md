# Gamiary UI/UX Pro Max 设计规范

## 1. 设计系统 (Design System)

### 颜色 (Colors)
基于海外硬核游戏平台风格 (Overseas Gaming + Dark Mode)：
- **背景 (Background)**: `#0B0F19` (深邃海军蓝黑 - 海洋深处般的沉浸感，常用于Steam/Epic)
- **主色调 (Primary)**: `#0EA5E9` (高能电竞蓝/Neon Blue - 用于主标题和突出元素，脱离AI常用紫)
- **辅色调 (Secondary)**: `#38BDF8` (高亮亮青色 - 用于次要徽章、悬停状态)
- **点缀/强调色 (CTA)**: `#EF4444` (警报红 - 用于警示、主要行动按钮，与冷色调形成强烈撞色)
- **文本 (Text)**: `#F8FAFC` (Slate 50 - 干净利落的高对比度正文)
- **边框 (Border)**: `#1E293B` (深色灰蓝 - 用于隔离层级)

### 字体 (Typography)
- **标题字体 (Heading)**: `Russo One` (极具力量感和游戏感，支持展示级大写)
- **正文字体 (Body)**: `Chakra Petch` (现代科技感，具备高可读性)

### 间距 (Spacing)
- 采用 4px 基础网格系统 (e.g. `p-4`, `gap-6`, `m-2`)。
- 悬浮元素或内容卡槽与边缘保留充足呼吸感：建议边缘安全区 `16px(p-4)` 或 `24px(p-6)`。

### 圆角 (Border Radius)
- **硬核电竞感**: 按钮与徽章使用 `rounded-sm` 或 `rounded-md`，卡片类使用 `rounded-xl` 以形成张弛有度的视觉反差。

### 阴影 (Shadows / Effects)
- 暗黑模式减少常规大面积阴影，改用**局部发光 (Glow)**: `shadow-[0_0_15px_rgba(14,165,233,0.3)]` 或使用极细的内发光 `ring-1 ring-white/10` 来塑造体积感。

## 2. 层级结构指导 (Hierarchy Structure)

- **第一视觉落点**: CTA 按钮 (`#EF4444`) 或带有微发光动画的核心数据、主标题。
- **第二视觉落点**: 悬停（Hover）互动状态。互动区域必须通过轻微的 `#0EA5E9` 背景提亮和 `cursor-pointer` 提示。
- **降噪元素**: 面包屑、次要描述文本和装饰性图标应使用具有一定透明度的文字色 (`text-slate-400`），但不应低于 `#94A3B8` 以符合 WCAG 对比度要求。
- **空间隔离**: 避免板块之间完全黏合，使用细边框(`border-[#1E293B]`) 与透明度背景叠层（Layering）代替横线分割。

## 3. 组件规范 (Component Specs)

### 按钮 (Buttons)
- **交互**: 必定包含 `transition-colors duration-200` 平滑过渡，并保证最小触控区不小于 `44x44`（特别在移动端）。
- **动效**: 悬停时通过改变背景透明度或边框颜色反馈，**严禁使用会导致层结构偏移的 scale 放大变换**。
- **状态**: 异步操作时需自动进入 `disabled` 并展示 loading 状态，防止双击重复提交。

### 输入框 (Inputs)
- **视觉**: 静态时背景填充稍亮于底色（如 `bg-white/5`），边框透明；
- **聚焦**: 得到焦点时展示 `ring-2 ring-[#0EA5E9] outline-none` 以明确交互状态；必须搭配清晰的上方 `Label`，不单独依赖 placeholder。

### 卡片 (Cards)
- **基础**: 背景使用纯色或极其微弱的渐变（如 `bg-[#0B0F19]`），使用 `#1E293B` 或 `border-white/10` 细边框隔离。
- **悬停**: 卡片若可点击，需添加 `cursor-pointer` 并增加发光边框以凸显互动性。

### 徽章 (Badges)
- **形态**: 低透明度背景与高亮度文字组合（如 `bg-[#0EA5E9]/20 text-[#38BDF8]`），小号文字 (`text-xs`)。

### 链接 (Links)
- **样式**: 保留品牌色或下划线高亮，悬停时改变下划线粗细、颜色或增加光辉。

## 4. 深色 UI 中常见的陷阱 (Dark UI Pitfalls)

1. **不可见的投影 (Invisible Shadows)**: 在深色背景中尝试投射常规黑色阴影是无效的，反会使得层级变脏。**正解**: 使用带有透明度的浅色内发光 (`box-shadow: inset`) 或提亮底层元素的背景 `bg-white/10` 来表达 Z 轴高度。
2. **纯黑的误用 (`#000000`)**: 大量使用纯黑作为底色导致眼睛容易疲劳且无层级感。**正解**: 项目底层使用深色底（如 `#0B0F19`），而卡片向上叠加更亮阶梯的深灰阶。
3. **文本眩光晕影**: 纯白 (`#FFFFFF`) 文本在暗黑背景上因对比度过高会导致光晕效应，刺眼。**正解**: 降低一级白度，使用稍降一级的浅灰、浅蓝 (`#F8FAFC` 或 `#E2E8F0`) 作为正文。
4. **霓虹滥用**: 在所有元素上增加霓虹荧光发光，导致视觉找不到确切聚焦点。**正解**: 霓虹高亮仅限制于最重要的 CTA 按钮或活跃状态的 Tab，克制使用。
